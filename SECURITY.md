# Security policy

## Policy scope

This policy applies to the current `main` branch and the live GitHub Pages deployment. Older commits and forks are outside its scope.

## Report a security issue privately

Please use [GitHub private vulnerability reporting](https://github.com/llody9977/vulnsignal/security/advisories/new). Do not open a public issue for a suspected security vulnerability.

Include the affected file or workflow, the security impact, clear reproduction steps, and any suggested mitigation. Please do not include secrets or personal data in the report. Reports are reviewed when time permits, and I may not reply to every report. This personal project does not promise an acknowledgement, response, remediation or disclosure timeline, and it does not offer a bug bounty.

Issues in CVE, NVD, CISA KEV, FIRST EPSS, CWE, or a publisher's disclosure feed should normally be reported to that source owner. A source-data issue becomes a VulnSignal security issue when the dashboard or pipeline handles it in a way that could compromise this repository, its GitHub Pages deployment, or its users.
